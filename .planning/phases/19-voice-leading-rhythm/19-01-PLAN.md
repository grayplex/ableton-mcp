Create `MCP_Server/theory/voicing.py` with the following functions:

**`_has_parallel_motion(prev_midis, next_midis)`** (private helper):
- For each pair of voices (i, j) where j > i, compute `(prev_midis[j] - prev_midis[i]) % 12` and `(next_midis[j] - next_midis[i]) % 12`
- If both intervals are the same AND both are in `(0, 7)` (unison/octave or perfect 5th):
  - Check both voices moved: `motion_i = next_midis[i] - prev_midis[i]`, `motion_j = next_midis[j] - prev_midis[j]`
  - If `motion_i != 0` and `motion_j != 0` and `(motion_i > 0) == (motion_j > 0)` => return `True`
- Return `False` if no parallels found
- Operates on the minimum length of the two arrays when sizes differ (per Pitfall 1 in RESEARCH.md)

**`voice_lead_chords(source_midis, target_midis)`** (public):
- Per D-02: inputs are MIDI pitch arrays (list[int]), NOT chord name strings
- Per D-03: returns list of rich note objects `[{"midi": int, "name": str}, ...]`
- Algorithm:
  1. Sort source: `source_sorted = sorted(source_midis)`
  2. Compute center: `center = sum(source_sorted) / len(source_sorted)`
  3. For each target pitch, generate candidate placements near center: `pc = m % 12`, `base = int(center) - (int(center) % 12) + pc`, try `[base-12, base, base+12]`, pick closest to center
  4. For chords <= 5 notes, try all `itertools.permutations(candidates)`:
     - Sort each permutation, compute cost = sum of `abs(source - target)` for paired voices
     - Filter out permutations where `_has_parallel_motion(source_sorted, sorted_perm)` returns True (only when same number of voices)
     - Pick permutation with minimum cost from filtered set
  5. If ALL permutations have parallels (per Pitfall 2), fall back to minimum cost without filter
  6. For chords > 5 notes, use sorted candidates directly
  7. Convert result to note objects: `[_make_note_obj(_midi_to_pitch(m)) for m in best_perm]`
- Import `_make_note_obj` and `_midi_to_pitch` from `MCP_Server.theory.chords`
- Import `itertools.permutations` from stdlib

**`voice_lead_progression(key, numerals, scale_type="major", octave=4)`** (public):
- Per D-04: same interface as `generate_progression` (key + Roman numeral sequence)
- Import `_resolve_numeral_to_chord` from `MCP_Server.theory.progressions`
- Algorithm:
  1. Resolve each numeral via `_resolve_numeral_to_chord(numeral, key, scale_type, octave)`
  2. First chord: use as-is from resolution
  3. Each subsequent chord: extract MIDI pitches from previous result, extract MIDI pitches from current resolution, call `voice_lead_chords(prev_midis, curr_midis)` to get re-voiced notes
  4. Build chord object with numeral, root, quality from resolution + voice-led notes
  5. Return `{"key": key, "scale_type": scale_type, "chords": [chord_objects]}`
- Raises `ValueError` if numerals is empty

- `from MCP_Server.theory.chords import _make_note_obj, _midi_to_pitch` at top

- `"description"`: str
- `"time_signature"`: `"4/4"`

Tone references per D-06: `"root"`, `"3rd"`, `"5th"`, `"7th"`, `"octave"`

Minimum patterns (at least these; add more to reach ~15-20):
1. **Arpeggios (4-5):** `arpeggio_up` (ascending root-3rd-5th-octave in 8th notes across 4 beats), `arpeggio_down` (descending), `arpeggio_alternating` (up then down), `arpeggio_broken` (root-5th-3rd-octave pattern)
2. **Bass lines (3-4):** `bass_root_fifth` (root on 1, fifth on 3, quarter notes), `bass_walking` (root-3rd-5th-octave in quarter notes), `bass_octave` (root and octave alternating on 8ths)
3. **Comping (3-4):** `comping_block` (all chord tones on beats 1 and 3, half notes), `comping_syncopated` (chord hits on 1, and-of-2, 4), `comping_quarter` (all chord tones on every beat)
4. **Strumming (3-4):** `strumming_down` (all tones on each beat as 8th notes), `strumming_up_down` (alternating root-only and full chord), `strumming_folk` (bass on 1-3, full on 2-4)

Each step's beat positions should be within 0.0 to 3.75 (4/4 time = 4 beats, last 16th note at 3.75). Velocities typically 80-110.

- For any unrecognized tone reference, fall back to `chord_midis[0]`

- Return format: `[{"name": str, "category": str, "style": str, "description": str, "time_signature": str, "step_count": int}, ...]`

**`apply_rhythm_pattern(chord_midis, pattern_name, start_beat=0.0, duration=4.0)`** (public):
- Per D-08: output format EXACTLY matches `add_notes_to_clip`: `[{"pitch": int, "start_time": float, "duration": float, "velocity": int}]`
- Per D-09: accepts single chord (MIDI pitches list) + pattern name
- Per D-10: `start_beat` offset positions notes in clip timeline
- Per D-11: `duration` in beats, pattern plays once within that duration
- Algorithm:
  1. Look up pattern from `RHYTHM_CATALOG` by name. Raise `ValueError` if not found
  2. Sort chord_midis ascending
  3. For each step in pattern["steps"]:
     - If `step["beat"] >= duration`: skip (per Pitfall 3)
     - Resolve pitch: `_resolve_chord_tone(step["tone"], sorted_midis)`
     - Compute note duration: `min(step["duration"], duration - step["beat"])`
     - Append `{"pitch": pitch, "start_time": start_beat + step["beat"], "duration": note_dur, "velocity": step["velocity"]}`
  4. Return the notes list
