    **Step 2: Update _build_plan_sections in MCP_Server/tools/plans.py**

    Change the function signature to accept override params:
    ```python
    def _build_plan_sections(
        sections: list[dict],
        section_bar_overrides: dict | None = None,
        add_sections: list[dict] | None = None,
        remove_sections: list[str] | None = None,
    ) -> tuple[list[dict], list[str]]:
    ```

    Returns a tuple: `(plan_sections, warnings)` where warnings is a list of warning strings.

    Implementation order (per D-09):

    1. `sections = copy.deepcopy(sections)` (already done in Plan 01)

    2. **Remove sections**: If remove_sections is not None, build a set of names to remove. For each name in remove_sections, check if it exists in sections; if not, append to warnings list: `f"remove_sections: '{name}' not found in blueprint"`. Filter sections list: `sections = [s for s in sections if s["name"] not in remove_set]`.

    3. **Add sections**: If add_sections is not None, for each entry in add_sections:
       - Check for duplicate name: if entry["name"] matches any existing section name, this is an error (not a warning). Set an error flag and return early -- the caller (generate_production_plan) will return format_error.
       - Find anchor index: `next((i for i, s in enumerate(sections) if s["name"] == entry["after"]), None)`
       - If anchor not found: append warning `f"add_sections: anchor '{entry['after']}' not found, skipping '{entry['name']}'"` and skip this entry.
       - Otherwise: `sections.insert(idx + 1, {"name": entry["name"], "bars": entry["bars"], "roles": [], "energy": 5})`
       - Note: added sections do NOT get a "transition_in" key (no blueprint data for custom sections).

    4. **Resize sections**: If section_bar_overrides is not None, for each name, count in section_bar_overrides.items():
       - Find the section by name. If not found, append warning `f"section_bar_overrides: '{name}' not found in blueprint"`.
       - Otherwise set `s["bars"] = count`.

    5. **Calculate bar_start**: same cumulative logic as Plan 01 -- bar=1, each section gets bar_start=bar, bar += bars.

    6. **Build output entries**: same as Plan 01 -- name, bar_start, bars, roles, and transition_in only if present.

    7. Return `(output_sections, warnings)`.

    **Step 3: Update generate_production_plan to pass overrides and handle warnings/errors**
