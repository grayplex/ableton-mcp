    1. Install ruff as dev dependency:
       ```bash
       uv add --dev ruff
       ```

    2. Add ruff configuration to pyproject.toml (append these sections):
       ```toml
       [tool.ruff]
       line-length = 100
       target-version = "py311"

       [tool.ruff.lint]
       select = [
           "E",   # pycodestyle errors
           "F",   # pyflakes
           "W",   # pycodestyle warnings
           "I",   # isort (import ordering)
           "B",   # flake8-bugbear (common mistakes)
           "UP",  # pyupgrade (modernize syntax)
       ]
       ignore = [
           "E501",  # line too long -- handled by line-length setting
           "B905",  # zip() without strict= -- not needed for this project
       ]

       [tool.ruff.lint.per-file-ignores]
       "__init__.py" = ["F401"]  # unused imports OK in __init__.py (re-exports)
       "tests/**/*.py" = ["B011"]  # assert False OK in tests
       "AbletonMCP_Remote_Script/**/*.py" = [
           "UP035",  # deprecated import -- _Framework uses old patterns
       ]

       [tool.ruff.lint.isort]
       known-first-party = ["MCP_Server", "AbletonMCP_Remote_Script"]
       ```

    3. Run ruff check and fix auto-fixable issues:
       ```bash
       uv run ruff check MCP_Server/ AbletonMCP_Remote_Script/ tests/ --fix
       ```

    4. Manually fix any remaining lint errors that cannot be auto-fixed. Common fixes:
       - Import ordering (ruff will auto-fix)
       - Unused imports (remove or add noqa if intentional re-export)
       - pyupgrade suggestions (super() instead of super(Class, self))
       - bugbear issues (mutable default arguments, etc.)

    5. Run ruff format to standardize code style:
       ```bash
       uv run ruff format MCP_Server/ AbletonMCP_Remote_Script/ tests/
       ```

    6. Verify clean run:
       ```bash
       uv run ruff check MCP_Server/ AbletonMCP_Remote_Script/ tests/
       ```
       Must exit 0 with no errors.

    NOTE: The AbletonMCP_Remote_Script/__init__.py imports `from _Framework.ControlSurface import ControlSurface` which is only available in Ableton's runtime. ruff will flag this as an unresolved import (F401/E302). Add per-file-ignore or noqa comment for this specific import. Similarly, handler mixin modules may reference self._song etc. which are ControlSurface attributes -- ruff won't flag these since they're instance attribute access.
