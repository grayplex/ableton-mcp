**File 1: `AbletonMCP_Remote_Script/handlers/routing.py`**

Module docstring: `"""Routing handlers: get/set input and output routing types for all track types."""`

Imports:
```python
from AbletonMCP_Remote_Script.registry import command
from AbletonMCP_Remote_Script.handlers.tracks import _resolve_track
```

Class `RoutingHandlers` with 4 methods:

1. `@command("get_input_routing_types")` -- `_get_input_routing_types(self, params)`:
   - Extract `track_type = params.get("track_type", "track")`, `track_index = params.get("track_index", 0)`
   - Resolve track via `_resolve_track(self._song, track_type, track_index)`
   - Build `available` list: `[rt.display_name for rt in track.available_input_routing_types]`
   - Get `current`: `track.input_routing_type.display_name`
   - Return dict: `{"track_name": track.name, "track_type": track_type, "current": current, "available": available}`
   - Add track_index to result if track_type != "master"
   - Wrap in try/except, `self.log_message(f"Error getting input routing types: {e}")`, raise

2. `@command("set_input_routing", write=True)` -- `_set_input_routing(self, params)`:
   - Extract `track_type`, `track_index`, `routing_name = params.get("routing_name")`
   - Validate routing_name is not None: `raise ValueError("routing_name parameter is required")`
   - Resolve track via `_resolve_track`
   - Capture `previous = track.input_routing_type.display_name`
   - Case-insensitive match: iterate `track.available_input_routing_types`, compare `rt.display_name.lower() == routing_name.lower()`
   - On match: `track.input_routing_type = rt`
   - Build result: `{"previous": previous, "current": track.input_routing_type.display_name, "track_name": track.name, "track_type": track_type}`
   - Add track_index if track_type != "master"
   - If no match found: `available = [rt.display_name for rt in track.available_input_routing_types]` then `raise ValueError(f"Routing type '{routing_name}' not found. Available: {available}")`
   - Wrap in try/except, log, raise

3. `@command("get_output_routing_types")` -- `_get_output_routing_types(self, params)`:
   - Same pattern as get_input_routing_types but uses `available_output_routing_types` and `output_routing_type`
   - Same response shape: `{"track_name", "track_type", "current", "available"}` plus conditional track_index

4. `@command("set_output_routing", write=True)` -- `_set_output_routing(self, params)`:
   - Same pattern as set_input_routing but uses `available_output_routing_types` and `output_routing_type`
   - Same response shape: `{"previous", "current", "track_name", "track_type"}` plus conditional track_index

**File 2: `AbletonMCP_Remote_Script/handlers/audio_clips.py`**

Module docstring: `"""Audio clip handlers: get and set audio-specific clip properties (pitch, gain, warping)."""`

Imports:
```python
from AbletonMCP_Remote_Script.handlers.clips import _resolve_clip_slot
from AbletonMCP_Remote_Script.registry import command
```

Class `AudioClipHandlers` with 2 methods:

1. `@command("get_audio_clip_properties")` -- `_get_audio_clip_properties(self, params)`:
   - Extract `track_index = params.get("track_index", 0)`, `clip_index = params.get("clip_index", 0)`
   - Resolve: `clip_slot, track = _resolve_clip_slot(self._song, track_index, clip_index)`
   - Guard no clip: `if not clip_slot.has_clip: raise Exception("No clip in slot")`
   - `clip = clip_slot.clip`
   - Guard MIDI clip: `if not clip.is_audio_clip: raise ValueError(f"Clip at track {track_index} slot {clip_index} is a MIDI clip, not audio. Audio clip properties only apply to audio clips.")`
   - Build result dict:
     ```python
     result = {
         "track_name": track.name,
         "clip_name": clip.name,
         "pitch_coarse": clip.pitch_coarse,
         "pitch_fine": clip.pitch_fine,
         "gain": clip.gain,
         "gain_display": clip.gain_display_string,
         "warping": clip.warping,
     }
     ```
   - Bonus: try to read warp_mode (Claude's Discretion): `try: result["warp_mode"] = str(clip.warp_mode) except Exception: pass`
   - Return result
   - Wrap in try/except, log, raise

2. `@command("set_audio_clip_properties", write=True)` -- `_set_audio_clip_properties(self, params)`:
   - Extract: `track_index`, `clip_index`, `pitch_coarse = params.get("pitch_coarse", None)`, `pitch_fine = params.get("pitch_fine", None)`, `gain = params.get("gain", None)`, `warping = params.get("warping", None)`
   - Resolve clip_slot and track, guard no clip, guard MIDI clip (same as get)
   - Build `changes = []` list
   - If pitch_coarse is not None:
     - Validate: `if not (-48 <= pitch_coarse <= 48): raise ValueError(f"pitch_coarse {pitch_coarse} out of range (-48 to 48 semitones). Current value: {clip.pitch_coarse}")`
     - Capture prev, set, append to changes: `{"property": "pitch_coarse", "previous": prev, "current": clip.pitch_coarse}`
   - If pitch_fine is not None:
     - Validate: `if not (-500 <= pitch_fine <= 500): raise ValueError(f"pitch_fine {pitch_fine} out of range (-500 to 500). Current value: {clip.pitch_fine}")`
     - Capture prev, set, append to changes
   - If gain is not None:
     - Validate: `if not (0.0 <= gain <= 1.0): raise ValueError(f"gain {gain} out of range (0.0 to 1.0). Current value: {clip.gain} ({clip.gain_display_string})")`
     - Capture prev and prev_display = clip.gain_display_string before setting
     - Set clip.gain = gain
     - Append: `{"property": "gain", "previous": prev, "previous_display": prev_display, "current": clip.gain, "current_display": clip.gain_display_string}`
   - If warping is not None:
     - Capture prev, set clip.warping = warping
     - Append: `{"property": "warping", "previous": prev, "current": clip.warping}`
   - Return: `{"track_name": track.name, "clip_name": clip.name, "changes": changes}`
   - Wrap in try/except, log, raise

Note on gain: Per CONTEXT.md the user wanted "dB values directly" but per RESEARCH.md the API uses normalized 0.0-1.0. The research recommends accepting normalized values and including gain_display_string in responses (Claude's Discretion). Follow this approach -- accept 0.0-1.0 for gain, always include gain_display in responses.
