From AbletonMCP_Remote_Script/handlers/tracks.py (_get_track_info, lines 467-562):
The handler resolves a track via _resolve_track, then builds a result dict. The crash is in the
"common fields" block (lines 482-491):

```python
# Common fields for all track types
result = {
    "name": track.name,
    "type": type_str,
    "color": color_name,
    "volume": track.mixer_device.volume.value,
    "panning": track.mixer_device.panning.value,
    "mute": track.mute,        # CRASHES on master track — no .mute attribute
    "solo": track.solo,        # CRASHES on master track — no .solo attribute
}
```

The arm field just below is already correctly guarded:
```python
# Arm state (only for tracks that support it)
if hasattr(track, "can_be_armed") and track.can_be_armed:
    result["arm"] = track.arm
```

The fix pattern to follow — use hasattr before accessing optional attributes:
```python
if hasattr(track, "mute"):
    result["mute"] = track.mute
if hasattr(track, "solo"):
    result["solo"] = track.solo
```
