  <name>Task 1: Add track CRUD and color handlers to TrackHandlers mixin</name>
  <files>AbletonMCP_Remote_Script/handlers/tracks.py, MCP_Server/connection.py</files>
  <action>
Extend `TrackHandlers` class in `AbletonMCP_Remote_Script/handlers/tracks.py` with the following new handler methods. Follow the existing pattern: `@command("wire_name", write=True)` for state-modifying, `@command("wire_name")` for read-only. All methods follow `def _method(self, params)` signature.

**1. COLOR_NAMES dict (module-level constant):**
Define a `COLOR_NAMES` dict mapping ~70 friendly color name strings to indices 0-69. Use descriptive snake_case names covering the full Ableton palette (e.g., "salmon", "dark_red", "red", "light_orange", "orange", "gold", "yellow", "lime", "green", "mint", "cyan", "sky", "blue", "indigo", "purple", "violet", "magenta", "pink", "hot_pink", "cream", "peach", "tan", "brown", "dark_brown", etc. with light/dark/medium variants). Also define `COLOR_INDEX_TO_NAME = {v: k for k, v in COLOR_NAMES.items()}` for reverse lookup.

**2. create_audio_track (write=True):**
- Param: `index` (default -1)
- Call `self._song.create_audio_track(index)`
- Return `{"index": new_track_index, "name": track.name, "type": "audio"}`
- Same pattern as existing `_create_midi_track`

**3. create_return_track (write=True):**
- No index param (API always appends)
- Call `self._song.create_return_track()`
- new_index = len(self._song.return_tracks) - 1
- Return `{"index": new_index, "name": track.name, "type": "return"}`

**4. create_group_track (write=True):**
- Params: `index` (default -1), optional `track_indices` (list of ints)
- If no `track_indices`: create empty MIDI track at index (serves as group header). Note: this creates a plain track since the API has no direct group creation. Return `{"index": new_index, "name": track.name, "type": "group", "note": "Created as empty track. Add child tracks to make it a group."}`
- If `track_indices` provided: Attempt to group tracks programmatically. Try selecting tracks via `self._song.view.selected_track` and using internal grouping. If that fails, fall back to creating an empty track and documenting the limitation in the response.
- Research confidence on group creation is LOW. Implement best-effort with clear error messaging.
- Return includes `{"index", "name", "type", "grouped_track_indices"}` when grouping.

**5. delete_track (write=True):**
- Params: `track_type` (default "track"), `track_index`
- If track_type == "master": raise ValueError("Cannot delete the master track")
- If track_type == "return": validate bounds on `self._song.return_tracks`, capture name/type/index, call `self._song.delete_return_track(track_index)`
- Else: validate bounds on `self._song.tracks`, capture name/type/index, call `self._song.delete_track(track_index)`
- Return `{"deleted": {"name", "type", "index"}}`

**6. duplicate_track (write=True):**
- Params: `track_index`, optional `new_name`
- Validate bounds on `self._song.tracks`
- Call `self._song.duplicate_track(track_index)`
- New track at `track_index + 1`. If `new_name` provided, set `new_track.name = new_name`
- Determine type: "midi" if `has_midi_input`, "audio" if `has_audio_input`, "group" if `is_foldable`
- Return `{"index": track_index + 1, "name": new_track.name, "type": type_str}`

**7. set_track_color (write=True):**
- Params: `track_index`, `color` (string), optional `track_type` (default "track")
- Look up track using track_type/track_index pattern (regular, return, master)
- Validate color name against COLOR_NAMES. If not found: `raise ValueError(f"Unknown color '{color}'. Valid colors: {sorted_valid_list}")`
- Set `track.color_index = COLOR_NAMES[color]`
- Return `{"index": track_index, "name": track.name, "color": color}`

**8. set_group_fold (write=True):**
- Params: `track_index`, `folded` (bool)
- Get track from `self._song.tracks[track_index]`
- Validate `track.is_foldable`, raise ValueError if not
- Set `track.fold_state = 1 if folded else 0`
- Return `{"index": track_index, "name": track.name, "folded": folded}`

**Also update the existing `_create_midi_track` handler to include `"type": "midi"` in its response** for consistency across all create_* responses.

**Update MCP_Server/connection.py:**
Add all new write command wire names to the `_WRITE_COMMANDS` frozenset: `"create_return_track"`, `"create_group_track"`, `"delete_track"`, `"duplicate_track"`, `"set_track_color"`, `"set_group_fold"`. Note: `"create_audio_track"` is already present.

  <files>AbletonMCP_Remote_Script/handlers/tracks.py</files>

**1. Enhanced _get_track_info:**
- Add `track_type` param (default "track") supporting "track", "return", "master"
- Track resolution:
  - "master": `track = self._song.master_track` (no index needed)
  - "return": validate bounds on `self._song.return_tracks`, get by index
  - "track": validate bounds on `self._song.tracks`, get by index (current behavior)
- Build response with type-appropriate fields:
  - **Common fields (all types):** name, type (str: "midi"/"audio"/"group"/"return"/"master"), color (friendly name from COLOR_INDEX_TO_NAME), volume (mixer_device.volume.value), pan (mixer_device.panning.value), mute, solo, devices list, clip_slots list (not for master/return that lack clip_slots -- guard with hasattr or try/except)
  - **Regular tracks only:** arm (guard with `track.can_be_armed`), is_grouped, group_track (index + name if is_grouped)
  - **Group tracks (is_foldable=True):** fold_state (as boolean), is_foldable=True
  - **Return tracks:** No arm, no clip_slots (return tracks have clip_slots but they're typically empty -- include them if present)
  - **Master track:** No arm, no clip_slots, no index
- Determine track type string: if track_type == "master" -> "master", if track_type == "return" -> "return", else check `is_foldable` -> "group", `has_midi_input` -> "midi", `has_audio_input` -> "audio"
- Include color as friendly name from COLOR_INDEX_TO_NAME (fall back to "unknown_{index}" if not in map)

**2. New _get_all_tracks (read command):**
- `@command("get_all_tracks")` (read-only, no write=True)
- No params needed

```python
    "return_tracks": [{"index": i, "name": t.name, "type": "return", "color": color_name} for each return track],
    "master_track": {"name": master.name, "type": "master", "color": color_name}

```
- Type detection: same logic as get_track_info (is_foldable -> group, has_midi_input -> midi, has_audio_input -> audio)

**Important:** Keep the existing clip_slots and devices iteration logic from the current _get_track_info. Only extend it with the new fields and track type support.
