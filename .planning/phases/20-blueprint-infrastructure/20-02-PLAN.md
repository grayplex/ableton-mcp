    **Step 1: Create MCP_Server/genres/catalog.py**

    Three public functions + private discovery:

    ```python
    """Genre catalog: auto-discovery, alias resolution, and subgenre merge."""

    import importlib
    import logging
    import pkgutil
    from typing import Any, Dict, List, Optional

    import MCP_Server.genres as genres_package
    from MCP_Server.genres.schema import validate_blueprint

    logger = logging.getLogger("AbletonMCPServer")

    # Module-level registry populated on first import
    _registry: Dict[str, dict] = {}
    _alias_map: Dict[str, str] = {}
    _subgenres: Dict[str, Dict[str, dict]] = {}
    _initialized = False
    ```

    **Private function `_discover_genres()`:**
    - Uses `pkgutil.iter_modules(genres_package.__path__)` to find genre modules (per INFR-02)
    - Skips modules starting with `_` and modules named `catalog` or `schema` (infrastructure modules)
    - For each module: `importlib.import_module(f'MCP_Server.genres.{modname}')`
    - Expects each module to export `GENRE` (dict) and optionally `SUBGENRES` (dict of dicts)
    - Calls `validate_blueprint(genre_data)` — if ValueError raised, logs error and skips that genre (per D-08: fail per-genre, not per-server)
    - Populates `_registry[genre_id] = genre_data` where genre_id = `genre_data["id"]`
    - Populates `_subgenres[genre_id] = subgenres_data` if SUBGENRES exported
    - Builds `_alias_map` from each genre's `aliases` list + the canonical ID itself, plus each subgenre's `aliases` list mapped to `(genre_id, subgenre_id)` tuple
    - Alias normalization: `alias.lower().replace(' ', '_').replace('-', '_')`
    - Sets `_initialized = True`

    **Private function `_ensure_initialized()`:**
    - If not `_initialized`, call `_discover_genres()`
    - Called at top of every public function

    **Public function `list_genres() -> List[dict]`:**
    - Returns list of dicts, each with: `id`, `name`, `bpm_range`, `subgenres` (list of subgenre IDs)
    - Does NOT return full blueprint data — just metadata for listing

    **Public function `resolve_alias(alias: str) -> Optional[dict]`:**
    - Normalizes input: `alias.lower().replace(' ', '_').replace('-', '_')`
    - Checks `_alias_map` — if found, returns `{"genre_id": genre_id}` or `{"genre_id": genre_id, "subgenre_id": subgenre_id}`
    - If not found, returns None
    - Subgenre alias handling: when building alias map, subgenre aliases map to `{"genre_id": parent_id, "subgenre_id": sub_id}`. Store tuples in _alias_map: value is either `str` (genre only) or `tuple[str, str]` (genre + subgenre).

    **Public function `get_blueprint(genre_id: str, subgenre: Optional[str] = None) -> Optional[dict]`:**
    - First tries `resolve_alias(genre_id)` to handle aliases
    - Looks up base genre in `_registry`
    - If `subgenre` is None but resolved alias included a subgenre_id, use that
    - If subgenre requested: performs shallow merge (per INFR-04) — copies base dict, then for each key in subgenre override dict, replaces the base value entirely (shallow, not deep merge)
    - The merged result should NOT include a "subgenres" key (strip it if present from the copy)
    - Returns None if genre_id not found
