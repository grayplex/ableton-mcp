    Add these imports at top of file:

    Implementation of `get_genre_palette(ctx: Context, genre: str, key: str) -> str`:

    1. Call `get_blueprint(genre)` — if None, return `format_error("Genre not found", detail=f"'{genre}' is not a registered genre or alias", suggestion="Use list_genre_blueprints to see available genres")`

    2. Extract harmony section: `harmony = blueprint["harmony"]`

    3. Resolve scales — for each scale_name in `harmony["scales"]`:
       - If `scale_name in SCALE_CATALOG`: append `f"{key} {scale_name}"` to resolved_scales
       - Else: append scale_name to unresolved list

    4. Resolve chord_types — for each quality in `harmony["chord_types"]`:
       - If `quality in _QUALITY_MAP`: append `f"{key}{quality}"` to resolved_chords
       - Else: append quality to unresolved list

    5. Resolve progressions — for each numeral_list in `harmony["common_progressions"]`:
       - Determine scale_type heuristic: if first numeral is lowercase (e.g., "i", "ii"), use "natural_minor"; if uppercase (e.g., "I", "II"), use "major"
       - The lowercase check: `numerals[0][0].islower()` after stripping 'b'/'#' prefix. Specifically: strip leading 'b' and '#' chars, then check if first remaining char is lowercase
       - Try `generate_progression(key, numeral_list, scale_type=determined_type)`
       - Extract chord names from result: `[f"{c['root']}{c['quality']}" for c in result["chords"]]`
       - On ValueError: append `{"numerals": numeral_list, "error": str(e)}` to unresolved list, skip this progression

    6. Build result dict:
       ```python
       result = {
           "genre": blueprint["id"],
           "key": key,
           "scales": resolved_scales,
           "chord_types": resolved_chords,
           "progressions": resolved_progressions,
       }
       if unresolved:
           result["unresolved"] = unresolved
       ```

    7. Return `json.dumps(result)`. Wrap entire body in try/except returning `format_error("Failed to get genre palette", detail=str(e), suggestion="Check genre and key are valid")`.

    The `@mcp.tool()` docstring should be:
    ```
    Get key-resolved harmonic palette for a genre. Returns chord names, scale names, and common progressions resolved in the given key using the theory engine.

    Parameters:
    - genre: Genre name or alias (e.g., "house", "techno", "deep house")
    - key: Musical key root (e.g., "C", "F#", "Bb")
    ```
