  <action>
    Create `scripts/bootstrap_catalog.py` per D-01. The script:

    1. **Module docstring** explaining purpose, prerequisites (Ableton Live running, Remote Script active, session with all 12 target devices loaded on tracks), and usage: `python -m scripts.bootstrap_catalog`

    2. **TARGET_DEVICES list** -- the 12 display names:
       ```python
       TARGET_DEVICES = [
           "EQ Eight", "Compressor", "Glue Compressor", "Drum Buss",
           "Multiband Dynamics", "Reverb", "Delay", "Auto Filter",
           "Gate", "Limiter", "Envelope Follower", "Utility",
       ]
       ```

    3. **KNOWN_CONVERSIONS dict** -- hand-curated conversion metadata for common parameter patterns. Keyed by parameter name substring patterns. Per Pitfall 6, the bootstrap captures raw data and conversions are added based on known audio DSP conventions:
       ```python
       KNOWN_CONVERSIONS = {
           # EQ frequency parameters
           "Frequency": {"type": "log", "natural_min": 20, "natural_max": 22050, "unit": "Hz"},
           # EQ/Compressor gain parameters
           "Gain": {"type": "linear_db", "natural_min": -15, "natural_max": 15, "unit": "dB"},
           # Threshold parameters
           "Threshold": {"type": "linear_db", "natural_min": -40, "natural_max": 0, "unit": "dB"},
           # Attack/Release time parameters
           "Attack": {"type": "log", "natural_min": 0.01, "natural_max": 1000, "unit": "ms"},
           "Release": {"type": "log", "natural_min": 0.01, "natural_max": 3000, "unit": "ms"},
           # Dry/Wet parameters
           "Dry/Wet": {"type": "linear", "natural_min": 0, "natural_max": 100, "unit": "%"},
           # Output/Volume/Makeup gain
           "Output Gain": {"type": "linear_db", "natural_min": -36, "natural_max": 36, "unit": "dB"},
           "Makeup": {"type": "linear_db", "natural_min": -15, "natural_max": 15, "unit": "dB"},
       }
       ```
       Note: This is a starting set. The match logic should check if any key in KNOWN_CONVERSIONS appears as a substring of the parameter name. Parameters that don't match get `conversion: None`.

    4. **`match_conversion(param_name: str) -> dict | None`** function that checks KNOWN_CONVERSIONS patterns against the parameter name and returns the conversion dict or None.

    5. **`bootstrap()` function** that:
       a. Calls `get_ableton_connection()` from `MCP_Server.connection`
       b. Calls `ableton.send_command("get_session_info")` to get all tracks
       c. For each track, calls `ableton.send_command("get_device_parameters", {"track_index": i, "device_index": j})` for each device on the track
       d. Matches each device's `device_name` against TARGET_DEVICES
       e. For matched devices, builds the catalog entry:
          - Key: `class_name` from the response
          - `display_name`: `device_name` from the response
          - `parameters`: list of dicts with `name`, `index`, `min`, `max`, `is_quantized`, `conversion` (from match_conversion)
       f. Validates that all 12 TARGET_DEVICES were found. If any missing, prints which are missing and exits with error code 1.
       g. Writes `MCP_Server/devices/catalog.py` preserving the ROLES list (reads existing file to extract ROLES, or hardcodes the known ROLES list)

    6. **File generation** -- the script writes catalog.py as formatted Python source. Use `repr()` or `pprint.pformat()` for the CATALOG dict, then append the ROLES list. Include the header comment `# Generated by scripts/bootstrap_catalog.py -- do not edit manually`.

    7. **`if __name__ == "__main__": bootstrap()`** entry point with try/except that prints connection errors clearly (e.g., "Could not connect to Ableton Live. Ensure the Remote Script is loaded and Ableton is running.").

    8. The script should also print a summary after writing: "Catalog written to MCP_Server/devices/catalog.py with {N} devices and {total_params} parameters."
