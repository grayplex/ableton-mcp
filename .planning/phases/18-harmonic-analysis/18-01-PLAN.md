**Function 1: `detect_key(notes)` (per D-01, D-02, D-03)**
- Input: `notes` list of dicts with `pitch`, `start_time`, `duration` keys
- Raise `ValueError("No notes provided for key detection")` if notes is empty
- Build `music21.stream.Stream`, append `music21.note.Note(midi=nd["pitch"])` with `quarterLength = nd.get("duration", 1.0)` for each note dict
- Call `s.analyze('key')` to get primary Key object
- Build candidates list, starting with primary result, then from `k.alternateInterpretations` until 3 total
- For each candidate: apply `_force_sharp()` to tonic Pitch before extracting name (research open question 1), use `round(max(0, correlationCoefficient), 4)` for confidence (pitfall 2: correlation range normalization)
- Each candidate dict: `{"key": str, "mode": str, "confidence": float}` — e.g., `{"key": "C", "mode": "major", "confidence": 0.8723}`
- Return list of up to 3 candidate dicts

**Function 2: `analyze_clip_chords(notes, resolution='beat', beats_per_bar=4)` (per D-04, D-05, D-06, D-07)**
- Input: `notes` list of dicts, `resolution` one of `'beat'`, `'half_beat'`, `'bar'`, `beats_per_bar` int (default 4, per pitfall 5)
- Raise `ValueError` if notes is empty or resolution not in valid set
- `QUANTIZE_WINDOW = 0.1` beats tolerance (discretion area, per research recommendation)
- Grid size: `{'beat': 1.0, 'half_beat': 0.5, 'bar': float(beats_per_bar)}[resolution]`
- Compute `max_time = max(nd["start_time"] + nd["duration"] for nd in notes)`
- Iterate grid points from 0.0 to max_time in grid_size steps
- For each grid point, collect pitches of notes whose `start_time` falls within `[grid_point - QUANTIZE_WINDOW, grid_point + grid_size)` (snap off-grid notes to nearest grid, per pitfall 3)
- Deduplicate collected pitches with `sorted(set(segment_pitches))`
- If 2+ unique pitches: call `identify_chord(unique_pitches)`, take first result, build segment dict with `{"beat": float, "chord": str, "quality": str, "root": str, "notes": [{"midi": int, "name": str}, ...], "confidence": float}` where notes come from identify_chord result's notes field (rich note objects per D-16)
- If exactly 1 pitch: report as `{"beat": float, "chord": null, "single_note": {"midi": int, "name": str}}` (per pitfall 4)
- If 0 pitches: skip segment (empty segments omitted, discretion area)
- Return list of segment dicts

**Function 3: `analyze_harmonic_rhythm(notes, resolution='beat', beats_per_bar=4, key=None)` (per D-08, D-09, D-10, D-11)**
- Input: same as analyze_clip_chords plus optional `key` string
- Call `analyze_clip_chords(notes, resolution, beats_per_bar)` to get chord segments
- Determine grid_size same way as analyze_clip_chords (needed for duration calculation)
- Merge consecutive segments with same chord name into spans: `{"chord": str, "start_beat": float, "duration": float}`
- If `key` is provided and timeline has chords (per D-10, D-11): extract chord names (skip null/single-note entries), call `analyze_progression(chord_names, key)`, merge Roman numeral from each analysis result into corresponding timeline entry as `"numeral"` field
- Compute stats: `total_chords` = len(timeline entries with non-null chord), `average_changes_per_bar` = total_chords / (max_time / beats_per_bar) rounded to 2 decimals, `most_common_duration` = mode of duration values
- Return `{"timeline": [...], "stats": {"total_chords": int, "average_changes_per_bar": float, "most_common_duration": float}}`
