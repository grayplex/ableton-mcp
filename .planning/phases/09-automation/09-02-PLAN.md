**Tool 1: `get_clip_envelope`**
```python
@mcp.tool()
def get_clip_envelope(
    ctx: Context,
    track_index: int,
    clip_index: int,
    device_index: int,
    parameter_name: str | None = None,
    parameter_index: int | None = None,
    chain_index: int | None = None,
    chain_device_index: int | None = None,
    sample_interval: float = 0.25,
) -> str:
```
Docstring: `"""Get automation envelope for a device parameter in a clip. Two modes: (1) without parameter_name/parameter_index, lists all parameters that have automation on the device; (2) with parameter_name or parameter_index, returns sampled breakpoint data for that parameter's envelope.\n\nBreakpoints are sampled at sample_interval beats (default 0.25 = 1/16th note). Response includes parameter min/max/value for context.\n\nParameters:\n- track_index: Index of the track\n- clip_index: Index of the clip slot\n- device_index: Index of the device on the track\n- parameter_name: Parameter name for detail mode (case-insensitive)\n- parameter_index: Parameter index for detail mode\n- chain_index: Chain index for devices inside rack chains\n- chain_device_index: Device index within a rack chain\n- sample_interval: Sampling interval in beats (default 0.25)"""`

Build params dict conditionally (same pattern as tools/devices.py): always include track_index, clip_index, device_index. Conditionally add parameter_name, parameter_index, chain_index, chain_device_index, sample_interval (only if != 0.25).
Send command `"get_clip_envelope"`. Return `json.dumps(result, indent=2)`.
Error suggestion: `"Verify clip and device exist with get_clip_info and get_device_parameters"`

```python
@mcp.tool()
def insert_envelope_breakpoints(

    parameter_name: str | None = None,
    parameter_index: int | None = None,
    chain_index: int | None = None,
    chain_device_index: int | None = None,
    step_length: float = 0.0,
) -> str:
```
Docstring: `"""Insert automation breakpoints into a clip envelope for a device parameter. Accepts a list of {time, value} pairs. Breakpoints merge with existing automation -- new breakpoints are added at specified times, existing breakpoints at other times are preserved.\n\nValues are clamped to the parameter's min/max range. step_length controls the duration of each step in beats (0.0 = point/instantaneous change).\n\nParameters:\n- track_index: Index of the track\n- clip_index: Index of the clip slot\n- device_index: Index of the device\n- breakpoints: List of {time: float, value: float} pairs\n- parameter_name: Parameter name (case-insensitive)\n- parameter_index: Parameter index\n- chain_index: Chain index for rack chain devices\n- chain_device_index: Device index within a rack chain\n- step_length: Duration of each step in beats (default 0.0 = point)"""`

Build params dict: always include track_index, clip_index, device_index, breakpoints. Conditionally add parameter_name, parameter_index, chain_index, chain_device_index, step_length (only if != 0.0).
Send command `"insert_envelope_breakpoints"`. Return `json.dumps(result, indent=2)`.
Error suggestion: `"Verify parameter exists with get_device_parameters. Check envelope exists with get_clip_envelope."`
