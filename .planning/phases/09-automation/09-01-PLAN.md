Implement 3 command handlers:

**1. `_get_clip_envelope(self, params)` -- registered as `@command("get_clip_envelope")`**

Parameters from params dict:
- `track_index` (int, default 0)
- `clip_index` (int, default 0)
- `device_index` (int, default 0)
- `parameter_name` (str | None, default None)
- `parameter_index` (int | None, default None)
- `chain_index` (int | None, default None)
- `chain_device_index` (int | None, default None)
- `sample_interval` (float, default 0.25)

Dual mode per CONTEXT decision:

**List mode** (both parameter_name and parameter_index are None):
- Resolve clip via `_resolve_clip_slot(self._song, track_index, clip_index)`. Raise if `not clip_slot.has_clip`.
- Check `clip.has_envelopes` -- if False, return `{"has_envelopes": False, "parameters_with_automation": []}` immediately.
- Resolve device via `self._resolve_device(params)` (inherits from DeviceHandlers).
- Iterate `device.parameters`. For each parameter, call `clip.automation_envelope(param)`. If result is not None, add `{"name": param.name, "index": i, "min": param.min, "max": param.max}` to list.
- Return `{"has_envelopes": True, "device_name": device.name, "parameters_with_automation": [list]}`.

**Detail mode** (parameter_name or parameter_index provided):
- Resolve clip, then device, then parameter using the same case-insensitive first-match lookup pattern from `_set_device_parameter` in devices.py. If `parameter_name` provided, iterate `device.parameters` with `p.name.lower() == parameter_name.lower()` for first match. If `parameter_index` provided, bounds-check and index directly. Raise ValueError/IndexError with available parameter names on failure.
- Call `envelope = clip.automation_envelope(param)`.
- If envelope is None: return `{"parameter_name": param.name, "device_name": device.name, "has_automation": False, "breakpoints": [], "min": param.min, "max": param.max, "value": param.value}`.
- If envelope is not None: sample `envelope.value_at_time(t)` at intervals of `sample_interval` from 0.0 to `clip.length` inclusive. Build breakpoints list as `[{"time": round(t, 4), "value": round(val, 6)}, ...]`. Return `{"parameter_name": param.name, "device_name": device.name, "has_automation": True, "sample_interval": sample_interval, "breakpoints": breakpoints, "min": param.min, "max": param.max, "value": param.value}`.

**2. `_insert_envelope_breakpoints(self, params)` -- registered as `@command("insert_envelope_breakpoints", write=True)`**

Parameters from params dict:
- `track_index` (int, default 0)
- `clip_index` (int, default 0)
- `device_index` (int, default 0)
- `parameter_name` (str | None, default None)
- `parameter_index` (int | None, default None)
- `chain_index` (int | None, default None)
- `chain_device_index` (int | None, default None)
- `breakpoints` (list of {"time": float, "value": float}, required)
- `step_length` (float, default 0.0)

Implementation:
- Resolve clip (must have clip), device, and parameter (same pattern as detail mode above). One of parameter_name or parameter_index must be provided -- raise ValueError if neither.
- Get envelope via `clip.automation_envelope(param)`.
- If envelope is None, raise ValueError with message: `f"Cannot create automation envelope for '{param.name}' on this clip. Try clearing the clip envelopes first and retry, or verify the parameter belongs to a device on the same track as the clip."`.
- Clamp each breakpoint value to `param.min` / `param.max` (same clamping pattern as `_set_device_parameter`). Track if any were clamped for warning.
- Loop through breakpoints and call `envelope.insert_step(bp["time"], bp["value"], step_length)` for each.
- After insertion, sample total breakpoints by reading `envelope.value_at_time()` at sample_interval=0.25 across clip.length to get a `total_breakpoints` count (number of unique sample points).
- Return: `{"parameter_name": param.name, "device_name": device.name, "breakpoints_inserted": len(breakpoints), "total_breakpoints": total_count}`. Add `"warning"` key if any values were clamped.

**3. `_clear_clip_envelopes(self, params)` -- registered as `@command("clear_clip_envelopes", write=True)`**

Parameters from params dict:
- `track_index` (int, default 0)
- `clip_index` (int, default 0)
- `device_index` (int | None, default None)
- `parameter_name` (str | None, default None)
- `parameter_index` (int | None, default None)
- `chain_index` (int | None, default None)
- `chain_device_index` (int | None, default None)

Dual mode per CONTEXT decision:

**Clear all mode** (device_index is None, AND parameter_name is None, AND parameter_index is None):
- Resolve clip (must have clip).
- Call `clip.clear_all_envelopes()`.
- Return `{"cleared": "all", "envelopes_cleared": "all"}`.

**Clear single parameter mode** (device_index provided, plus parameter_name or parameter_index):
- Resolve clip, device, parameter (same pattern).
- Call `clip.clear_envelope(param)`.
- Return `{"cleared": param.name, "device_name": device.name, "envelopes_cleared": 1}`.

All three handlers follow the try/except pattern with `self.log_message(f"Error ...: {e}")` then `raise`.
